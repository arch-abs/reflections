Q: When would you want to use a remote repository rather than keeping all your work local?


A: Incase our local branch is lost we can use remote as a backup.
Also having commits uploaded to remote makes it easier collaborating from distance as we reviewing of code.


Q: Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

A: Autommatic pulling might occur at times when not really required or intended.
Manual pulling gives us more control.


Q: Describe the differences between forks, clones, and branches. When would you use one instead of another?

A: Forking a is like cloning on to GitHub itself.
Cloning is like creating another copy.
Branching means a different route of commits.
